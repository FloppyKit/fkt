# Plan / Spec: v0.2 Taproot script-path (single-sig leaves)

**Date:** 2026-07-09  
**Product target:** `v0.2.0` when synthetic (+ optional real) fixtures green  
**Depends on:** v0.1.1 keypath freeze  
**Unlocks:** Offline Ark/Bark **exit-claim** style leaves (user key only)

---

## 1. Goal

Sign a PSBT input that spends Taproot via **script-path**:

```
witness stack:  [ schnorr_sig, tapscript, control_block ]
```

for **single-key leaves** of the form used by simple Tapscript and bark-style user leaves  
(e.g. `<xonly_pubkey> OP_CHECKSIG` with optional CSV/CLTV wrappers later).

## 2. Non-goals (v0.2)

- MuSig2 / multi-party leaves  
- Annex  
- Multiple concurrent leaf choices beyond “first leaf script we can sign”  
- Full bark protocol / ASP  
- Changing Ice Cold / Warm seed policy  

## 3. PSBT fields (BIP-371)

| Type | Hex | Role |
|------|-----|------|
| `PSBT_IN_TAP_LEAF_SCRIPT` | `0x15` | Key = control block; Value = script \|\| leaf_version |
| `PSBT_IN_TAP_BIP32_DERIVATION` | `0x16` | Path for internal / leaf key |
| `PSBT_IN_TAP_INTERNAL_KEY` | `0x17` | 32-byte x-only internal key |
| `PSBT_IN_TAP_MERKLE_ROOT` | `0x18` | 32-byte merkle root (script tree present) |

**v0.1 behavior:** presence of `0x18` → hard die.  
**v0.2 behavior:** if `0x15` present and key matches → **script-path sign**; if `0x18` without usable leaf → die/reject clearly.

## 4. Crypto

### 4.1 tapleaf_hash

```
tapleaf_hash = tagged_hash("TapLeaf", leaf_version || compact_size(script) || script)
```

`leaf_version` typically `0xc0` (BIP342 Tapscript).

### 4.2 BIP341 sighash (script-path)

Extend keypath preimage (epoch, hash_type, nVersion, nLockTime, sha_* caches):

- `spend_type = 0x02`  (ext_flag=1, no annex)  
- `input_index` (4 LE)  
- then: `tapleaf_hash` (32) \| `key_version` (1, =0) \| `codesep_pos` (4 LE, `0xffffffff` if none)

```
sighash = tagged_hash("TapSighash", preimage)
```

SIGHASH_DEFAULT (0x00) only in v0.2 unless already present.

### 4.3 Signature key

**Script-path uses the untweaked leaf private key** (no TapTweak for the signature itself).

Use `fkt_schnorr_sign` (not `fkt_schnorr_sign_taproot`).

### 4.4 Control block

Use control block bytes from PSBT key of `0x15` as final witness element (no recompute required if PSBT is well-formed). Optional v0.2.1: verify control block against internal key + merkle root.

### 4.5 Witness finalization

```
stack:  n=3
  item0: 64-byte schnorr (SIGHASH_DEFAULT: no sighash byte appended)
  item1: tapscript
  item2: control_block
```

Also empty final scriptSig (`0x07` empty) as today for segwit/taproot.

## 5. Code touch points (C89)

| File | Change |
|------|--------|
| `fkt_psbt.h` | `FKT_PSBT_IN_TAP_LEAF_SCRIPT 0x15`; state: merkle root, leaf script, leaf ver, control block, script_path flag |
| `fkt_psbt.c` | Parse `0x15`/`0x18`; **remove hard die** when script-path data complete; still die if tree with no leaf we can use |
| `fkt_sighash.h/.c` | `fkt_bip341_sighash_scriptpath(input_index, sighash)` |
| `fkt_signer.c` | Branch: if script_path → untweaked sign + 3-stack witness |
| Tests | Synthetic single-leaf PSBT + harness case `expect=sign` |

Constraints: ANSI C89, only libsecp256k1, secure zero of keys, no new deps.

## 6. Test plan

1. **Synthetic** (required for v0.2 tag): known mnemonic, single-leaf `<pk> OP_CHECKSIG`, build unsigned PSBT with 0x15/0x17/0x18, FKT signs, verify with secp or reference.  
2. **Reject**: merkle root without leaf → non-zero + clear message.  
3. **Regression**: v0.1.1 golden matrix still green (keypath unchanged).  
4. **Later:** real Sparrow/bark script-path export; Ark exit-claim fixture.
